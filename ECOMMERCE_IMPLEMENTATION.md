【app/ecommerce_data.py】
- 概要: ECサイトの商品データベース
- 内容:
  * Product クラス: 商品を表すデータクラス
    - product_id: 商品ID（ユニーク）
    - name: 商品名
    - category: カテゴリ（ファッション、家電、キッチン用品など）
    - price: 価格（円）
    - description: 商品説明
    - features: 特性リスト（例：防水、軽量など）
    - keywords: キーワード（検索用）
    - rating: 評価（0-5）
    - stock: 在庫数
  
  * PRODUCTS_DATABASE: 20+個のサンプル商品（実装時はDBから取得）
  
  * ヘルパー関数:
    - get_all_products(): 全商品を返す

"""

"""
【app/product_recommender.py】
- 概要: 商品検索とレコメンドロジック
- 主要関数:
  
  1. calculate_relevance_score()
     - 商品の関連度スコアを計算（0-100）
     - 考慮要因:
       * キーワードマッチング（最大50点）
         → 名前、説明、特性、キーワードで検索
       * 予算スコア（最大30点）
         → 予算内なら満点、外れたら減点
       * カテゴリボーナス（最大15点）
       * 在庫ボーナス（最大3点）
       * レーティングボーナス（最大2点）
  
  2. recommend_products()
     - ユーザーのニーズに基づいて商品をレコメンド
     - 入力: user_needs（テキスト）, budget, categories, top_n
     - 出力: (商品, スコア, 推奨理由)のリスト
     - 処理:
       1) テキストからキーワードを抽出
       2) 全商品をスコア計算
       3) スコアでソート
       4) 推奨理由を生成
  
  3. search_products()
     - キーワード検索機能
     - スコアリングロジック:
       * 名前でのマッチ: 50点
       * 説明でのマッチ: 30点
       * キーワードでのマッチ: 20点
       * 特性でのマッチ: 10点

"""

"""
【app/ecommerce_llm_service.py】
- 概要: LLMベースのニーズ解析
- 主要関数:
  
  1. parse_user_needs_with_llm(user_input)
     - LLMを使ってユーザー入力から詳細なニーズを抽出
     - 出力: {
         "keywords": [...],
         "budget_min": int,
         "budget_max": int,
         "categories": [...],
         "summary": str
       }
     - LLMがない場合: _fallback_parse_needs() で簡易解析
  
  2. _fallback_parse_needs(user_input)
     - LLM不要の簡易ニーズ解析
     - 正規表現を使って予算を抽出
     - キーワードから自動でカテゴリを判定
  
  3. format_recommendation_response()
     - レコメンド結果を見やすくフォーマット

"""

"""
【ecommerce_chatbot.py】

# 3. 関連度スコアの計算アルゴリズム
# ============================================================================

"""
関連度スコアの計算は、複数の要因を組み合わせた加重スコアリングシステムです。

例: 「防水で軽いジャケット」というニーズの場合

1. キーワード抽出: ["防水", "軽い", "ジャケット"]

2. 各商品のスコア計算（例: 「防水ワークジャケット」の場合）
   - キーワードマッチ: 
     * "防水" → 名前、特性に含まれる → 3/3 = 100% × 50点 = 50点
   - 予算スコア: 8,900円
     * 予算内 → 30点
   - カテゴリボーナス:
     * ファッションカテゴリ selected → 15点
   - 在庫ボーナス:
     * stock > 50 → 3点
   - レーティング:
     * rating = 4.5 → (4.5/5) × 2 = 1.8点
   
   合計: 50 + 30 + 15 + 3 + 1.8 = 99.8点

3. スコアが高い順にソート

"""

【ユーザビリティ】
- 自然言語入力でニーズを指定
- LLMなしでも簡易解析で動作
- サイドバーで細かくフィルタ可能
- 推奨理由を明確に表示

【テスト】
- 14個のユニットテストで主要機能をカバー
